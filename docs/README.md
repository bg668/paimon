## 3. 建议阅读路线

### 如果你是首次了解 paimonsdk

1. 先读 [paimonsdk 功能说明](./paimonsdk%20功能说明.md)，建立能力边界和运行时心智模型。
2. 再读 [paimonsdk 开发者参考](./paimonsdk%20开发者参考.md)，了解最小接入路径和关键 API。
3. 如需追溯实现背景，再补充阅读 [概要设计](./概要设计.md) 和 [详细设计](./详细设计.md)。

### 如果你准备开始接入

1. 从 [paimonsdk 开发者参考](./paimonsdk%20开发者参考.md) 的“快速开始”开始。
2. 再按需阅读“Agent 生命周期与核心方法”“事件与流式输出”“工具调用参考”“上下文与模型适配”。
3. 如需了解能力边界和当前未打通项，回看 [paimonsdk 功能说明](./paimonsdk%20功能说明.md) 中的“配置项说明”和“限制、当前边界与验证结论”。

### 如果你在做评审或发布检查

1. 优先阅读 [paimonsdk 功能说明](./paimonsdk%20功能说明.md)，确认对外表述与当前实现一致。
2. 再抽查 [paimonsdk 开发者参考](./paimonsdk%20开发者参考.md) 中的最小示例、配置表和错误处理章节，确认是否满足接入说明要求。
3. 如需核对设计落地情况，再对照 [详细设计](./详细设计.md) 查看差异。

## 5. 使用建议

- 对外发布时，建议把 [paimonsdk 功能说明](./paimonsdk%20功能说明.md) 视为能力边界文档，把 [paimonsdk 开发者参考](./paimonsdk%20开发者参考.md) 视为接入手册。
- 如需做二次封装或新增适配器，建议同时更新“功能说明”“开发者参考”和对应设计文档，避免实现与说明脱节。
